# Blank Meets Blank

A movie idea generator built on the classic Hollywood pitch format:

> _"Jaws meets 2001: A Space Odyssey."_

Type two movie titles into the blanks — or hit **Surprise Me** to drop in two
random well-known films, or just leave them empty — then hit **Pitch It** to
generate a brand-new fictional movie: a title, a one-sentence logline, and a full
synopsis, written by Claude.

## Stack

- **Next.js** (App Router) + **React 19**
- **TypeScript**
- **Tailwind CSS v4**
- **Claude** (Anthropic API) for the AI-written pitches, called from a secure
  server route — the API key never reaches the browser.
- No database, no auth. Local generation stays in as an automatic fallback.

## Run locally

```bash
npm install
cp .env.example .env.local   # then add your Anthropic API key (see below)
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Setting up the API key

The AI pitches are generated by Claude via the Anthropic API.

1. Create a key at
   [console.anthropic.com](https://console.anthropic.com/settings/keys).
2. Copy the example env file and paste your key in:
   ```bash
   cp .env.example .env.local
   ```
   ```
   # .env.local
   ANTHROPIC_API_KEY=sk-ant-...
   ```
3. Restart `npm run dev` so the new env var is picked up.

The key is read **only** on the server (`app/api/pitch/route.ts`) and is never
exposed to the browser. **No key? No problem** — if the variable is missing or
the AI call fails for any reason, the app automatically falls back to local
pitch generation, so it always works.

When deploying to **Vercel**, add `ANTHROPIC_API_KEY` under
**Project → Settings → Environment Variables**.

## How a pitch is generated

```
UI (page.tsx)
  └─ generatePitch()            lib/pitch.ts        ← the one seam the UI calls
       └─ POST /api/pitch       app/api/pitch/route.ts
            ├─ resolvePair()    lib/generator.ts    ← fill empty blanks w/ random films
            ├─ Claude API       (server-side, secret key)
            └─ on failure ──▶ composeLocalPitch()    lib/generator.ts (fallback)
  └─ on network failure ─────▶ generateLocalPitch()  lib/generator.ts (fallback)
```

## Project structure

```
app/
  layout.tsx          Root layout + fonts + metadata
  page.tsx            The poster UI (the only place the UI lives)
  globals.css         Tailwind import + theme + animations
  api/pitch/route.ts  Secure server route — calls Claude, falls back to local
lib/
  movies.ts           Built-in catalog of films + random/lookup helpers
  generator.ts        Local pitch generation (the AI-free fallback)
  pitch.ts            Client seam: calls /api/pitch, falls back to local
```

## Swapping or extending the generator

The UI depends on exactly one function: `generatePitch(input)` in `lib/pitch.ts`.
To change models, edit `MODEL` / the prompt in `app/api/pitch/route.ts`. To
change the offline behavior, edit `lib/generator.ts`. No component changes
required either way.
